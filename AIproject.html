このサイトを作るまで——HTML/CSSを触り始めて最初の2週間

2026-06-01  ·  プログラミング・AI

← Categories に戻る

「自分のサイトを持ちたい」という気持ちはずっとあった。 WordPressでブログを試したこともあったが、テーマのカスタマイズに限界を感じてやめた。 結局、ゼロから HTML / CSS を書いて作ることにした。

プログラミング経験はPythonを少し触った程度。Webは完全に初めてだった。 この記事は、最初の2週間で学んだこと・つまずいたこと・そろえたツールの記録だ。

最初に学んだこと

HTML はページの「骨格」、CSS は「見た目」

最初に理解すべき分業がこれだった。 HTMLで要素を並べ、CSSで色・大きさ・位置を決める。 この2つを混同したまま進むと、どこを直せばいいかわからなくなる。

ボックスモデルを先に理解する

CSSで一番最初に詰まるのがレイアウトだ。 要素には margin（外側の余白）→ border → padding（内側の余白）→ content という層がある。 これを図で理解してから触ると、「なぜかズレる」現象が激減した。

.card { padding: 1.5rem; /* 内側の余白 */ border: 1px solid #333; margin-bottom: 1rem; /* 外側の余白 */ }

Flexbox でほとんど並べられる

横並び・縦並び・中央揃えはほぼ Flexbox で解決できる。 display: flex を覚えるだけで レイアウトの9割が組めるようになった気がした。 Grid は「カード一覧」のような格子状レイアウトに使うと強力だとわかってきた。

つまずいたこと

スタイルが「効いていない」と思ったら、適用順の問題だった

CSSは後に書いたルールが優先される。さらに id は class より強い—— この「詳細度（specificity）」の概念を知らないまま進んでいたので、 なぜか書いたスタイルが反映されない現象が頻発した。

スタイルが効かないときはまずブラウザの DevTools（F12）で「どのルールが上書きしているか」を確認する。闇雲に !important を使うと後で詰む。

フォントが日本語だけ崩れる

英字フォントを指定したら日本語部分だけデフォルトのセリフ体になってしまった。 CSSの font-family はカンマ区切りで複数指定でき、 左から順にフォールバックされる。 日本語対応フォントを後ろに並べておくことで解決した。

font-family: 'Helvetica Neue', Arial, 'Hiragino Sans', 'Noto Sans JP', sans-serif;

スマホで崩れる

PCで完璧に見えていたレイアウトが、スマホで確認したら完全に崩れていた。 原因は2つ。①メタタグの viewport 指定がなかったこと、 ②固定幅（px）で組んでいたことだ。 以来、幅は % や max-width で指定し、 メディアクエリで画面幅ごとに調整するようにした。

<meta name="viewport" content="width=device-width, initial-scale=1.0"> は head の中に必ず入れる。これがないとスマホで縮小表示になる。

そろえたツール

環境を整えるだけで作業効率が大きく変わった。使っているツールをまとめて紹介する。

エディタ

VS Code

補完・ハイライト・拡張機能が豊富。Live Preview 拡張を入れるとファイルを保存するたびにブラウザが自動更新されて快適になった。

AI アシスタント

Claude Code

コードの質問・生成・デバッグに使っている。「このCSSがなぜ効かないか」を聞くと原因と修正案をセットで返してくれる。

デザイン参考

Dribbble / Awwwards

UIのアイデアが浮かばないときに眺める。「このデザインをどうコードで再現するか」を考える練習にもなっている。

カラー

Coolors

配色を決めるのが苦手なので、スペースキーでランダム生成してベースパレットを決めている。このサイトの黒×黄緑もここから。

フォント

Google Fonts

日本語フォント「Noto Sans JP」を採用。link タグ1行で読み込めるのが手軽で使いやすい。

バージョン管理

Git / GitHub

「前の状態に戻したい」が何度かあった。git init して以来、変更のたびにコミットする習慣がついた。

2週間やってみて

最初は「タグを覚えなきゃ」と思っていたが、実際は「作りたいものを決めて、わからないところを調べる」 繰り返しで自然と身についていった。 完璧に理解してから書こうとすると手が止まるので、 動くものを先に作って後から整えるほうが自分には合っていた。

次はJavaScriptを触り始めている。 インタラクションが加わると、サイトが一気に「動く」感じになって楽しい。 その記録もいつかここに書く予定だ。
